# 바이브코딩 치트키 15 — AI 조련 워크플로우

## 목적
바이브코딩으로 프로젝트를 시작부터 완성까지 이끄는 15단계 AI 조련 방법론. 5개 그룹으로 구성된 순차적 워크플로우로, AI를 "잘 쓰는 것"이 아닌 "잘 조련하는" 접근법.

## 단계

### 그룹 1: 시작 전 준비

1. **PRD 작성** (키 1)
   - 입력: 프로젝트 아이디어
   - 실행: 프로젝트 이름, 핵심 기능, 기술 스택, 페이지 구조를 문서화
   - 출력: AI가 이해할 수 있는 제품 요구사항 문서 (PRD)
   - 핵심: 코드 작성 전에 "설계도"를 먼저 보여줘야 AI가 올바른 방향으로 작업

2. **페르소나 부여** (키 3)
   - 입력: 프로젝트에 필요한 전문 분야
   - 실행: 프롬프트에 역할, 전문성, 코딩 스타일, 제약 조건을 명시
   ```
   너는 10년차 시니어 Next.js 개발자다.
   클린 코드와 타입 안정성을 최우선으로 하며,
   컴포넌트 재사용성을 중시한다.
   ```
   - 출력: 전문가 수준의 AI 응답 품질

3. **기술 스택 고정** (키 8)
   - 입력: 프로젝트 요구사항
   - 실행: 규칙 파일(CLAUDE.md / Cursor Rules)에 스택 명시
   ```
   2026 추천 스택: Next.js + Supabase + Tailwind CSS
   ```
   - 출력: 일관된 기술 스택 유지

### 그룹 2: AI와 소통

4. **컨텍스트 유지** (키 4)
   - 입력: 진행 중인 대화
   - 실행: (1) 규칙 파일 생성 (2) 대화 길어지면 요약 요청 (3) 한 번에 적게 시키기
   - 출력: AI의 맥락 이해도 유지

5. **한 번에 하나씩** (키 5)
   - 입력: 구현할 기능 목록
   - 실행: 1프롬프트 = 1기능, 파일 변경 최소화
   - 출력: 각 기능의 안정적 구현

6. **프롬프트 체이닝** (키 6)
   - 입력: 복잡한 기능 요구사항
   - 실행: 기능을 단계별로 분해하여 순차 요청
   ```
   예: 결제 기능
   Step 1: "결제 데이터 모델을 설계해 줘"
   Step 2: "결제 API를 만들어 줘"
   Step 3: "결제 UI를 만들어 줘"
   ```
   - 출력: 복잡한 기능의 안전한 구현

7. **에러 메시지 활용** (키 7)
   - 입력: 발생한 에러
   - 실행: 에러 메시지 + 운영 환경 + 시도한 것을 AI에게 전달
   - 출력: 30분 디버깅 → 30초 해결

### 그룹 3: 코드 관리

8. **구조 읽기** (키 8)
   - 입력: 프로젝트 코드베이스
   - 실행: 코드 한 줄씩 읽지 말고 폴더 구조와 데이터 흐름 파악
   - 출력: 프로젝트 전체 조감 능력

9. **도구 선택** (키 9)
   - 입력: 개발자 경험 수준
   - 실행: 초보자 → Cursor, 대규모 → Claude Code, 취향 → AntiGravity
   - 출력: 자신에게 맞는 AI 코딩 도구

### 그룹 4: 품질 향상

10. **AI 테스트 위임** (키 10)
    - 입력: 작성된 함수/컴포넌트
    - 실행: "이 함수에 대한 테스트 코드를 작성해 줘"
    - 출력: 자동화된 테스트 코드

11. **UI 레퍼런스 활용** (키 12)
    - 입력: 원하는 디자인
    - 실행: 스크린샷 첨부 + "이 레이아웃 참고해서 만들어 줘"
    - 출력: 모호한 프롬프트 대신 구체적 디자인 결과물

12. **AI 의심 습관** (키 13)
    - 입력: AI가 생성한 코드
    - 실행: 결과물 검증, 대안 탐색, 없는 라이브러리 확인
    - 출력: 신뢰할 수 있는 코드

### 그룹 5: 완성

13. **즉시 배포** (키 11)
    - 입력: 작동하는 최소 기능
    - 실행: 첫날부터 Vercel/AWS에 배포
    - 출력: 실제 URL = 강력한 동기 부여

14. **AI 리팩토링** (키 14)
    - 입력: 기능 3개 이상 완성된 코드
    - 실행: "코드 정리해 줘" — 중복 제거, 함수 분리, 네이밍 개선
    - 출력: 기술 부채 자동 청소

15. **문서화** (키 15)
    - 입력: 완성된 프로젝트
    - 실행: "개발 현황과 사용 방법을 README에 정리해 줘"
    - 출력: 3개월 뒤의 나를 위한 선물

## 팁
- 15개 키를 한 번에 다 적용할 필요 없음. 프로젝트 진행 단계에 맞춰 해당 그룹의 키 적용
- PRD와 페르소나는 프로젝트 시작 시 한 번만 설정하면 됨
- 프롬프트 체이닝은 모든 복잡한 기능에 적용 가능한 범용 기법
- "AI를 의심하라"는 모든 단계에서 적용되는 메타 원칙
