# 바이브코딩 UI/UX 프롬프트 모음

## 목적
바이브코딩에서 UI/UX 품질을 즉시 높일 수 있는 실전 프롬프트 모음. 투더제이 쇼츠에서 추출한 검증된 프롬프트 템플릿.

## 프롬프트 템플릿

### 1. AI 흔적 제거 — FontAwesome 교체
```
이모지 대신 FontAwesome 아이콘으로 사용할 것
```
AI가 만든 웹사이트의 가장 큰 특징은 이모지 과다 사용. 이 프롬프트 하나로 "AI가 만든 티"를 제거할 수 있다. 실제 디자이너들도 아이콘 라이브러리를 필수로 사용하는 실무 방식.

### 2. 디자인 스타일 지정
```
벤토디자인 적용
```
**❌ Bad**: "예쁘게 해줘" — 모호해서 결과물 불안정
**✅ Good**: "벤토디자인 적용" — 구체적 스타일 지정으로 완전히 다른 결과

단어 하나 차이가 완전히 다른 결과물을 만든다. "예쁜", "심플", "모던", "고급", "미니멀" 등의 형용사 조합도 실험해 볼 것.

### 3. UI 고도화 요소 추가
```
다음 UI/UX 요소를 적용해 줘:
- 스켈레톤 UI (로딩 상태)
- 3D 틸트 효과 (카드 인터랙션)
- 카운팅 애니메이션 (소셜프루프 숫자)
- 마그네틱 버튼 (호버 효과)
- 모바일 햄버거 메뉴
```

### 4. UI 레퍼런스 프롬프트
```
[스크린샷 첨부]
이 레이아웃 참고해서 만들어 줘
```
스크린샷 한 장이 100줄짜리 설명을 대체한다. 원하는 디자인의 스크린샷을 첨부하면 AI가 정확한 방향으로 작업.

### 5. 품질 관리 프롬프트
```
이 함수에 대한 테스트 코드를 작성해 줘
```
```
코드 정리해 줘 (중복 제거, 함수 분리, 네이밍 개선)
```

## 사용법
1. **초기 설정**: 프로젝트 규칙 파일에 "이모지 대신 FontAwesome 사용"을 고정
2. **디자인 요청 시**: 항상 구체적 스타일 키워드 사용 ("예쁘게" 금지)
3. **UI 고도화**: 기본 기능 완성 후 고도화 요소를 한 번에 추가
4. **레퍼런스 활용**: 마음에 드는 디자인을 스크린샷으로 첨부
