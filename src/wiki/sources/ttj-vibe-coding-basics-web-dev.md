# 투더제이 — 바이브코딩 입문자 웹 개발 기초 상식

## 개요
바이브 코딩 입문 Vol.2 (34분 31초). 웹 개발 기초 용어와 AI 관련 개념을 입문자 수준에서 정리. "대충 감을 잡고 만들면서 익히면 된다"는 학습 철학. 코딩 퀴즈 사이트 제공. 프로그래밍 언어의 발전사(어셈블리 → C → 웹언어 → 파이썬 → 바이브코딩)를 통해 현재 위치를 이해시킴.

## 핵심 내용

### 학습 철학
"대충하시는 게 가장 중요하다" — 기술 부채는 지금 걱정할 단계가 아니다. 그냥 대충 만들고 사용자가 생기고 수익이 생길 거 같을 때 그때 상황에 맞춰서 고민하면 된다. 완벽하게 외우는 게 아니라 대충 감을 잡고 만들면서 익히는 것이 바이브코딩의 학습법.

### 웹 기본 6가지

**1. 클라이언트와 서버**
클라이언트는 사용자의 기기(브라우저). 서버는 24시간 켜져 있는 컴퓨터. 클라이언트가 서버에 요청하면 서버가 응답하는 구조. "요청(Request) → 처리 → 응답(Response)"의 기본 흐름.

**2. 프론트엔드와 백엔드**
프론트엔드는 사용자가 보는 화면 부분. 백엔드는 보이지 않는 서버 쪽 처리. 프론트엔드 언어: HTML(구조), CSS(디자인), JavaScript(동작). 백엔드 언어: Node.js, Python, Java, PHP 등.

**3. HTML, CSS, JavaScript 역할**
HTML은 뼈대(구조). CSS는 피부(디자인/스타일). JavaScript는 근육(동작/인터랙션). 실제 예시로 네이버 메인 페이지를 분해해 보면 HTML 구조 위에 CSS 스타일이 적용되고, JavaScript로 햄버거 메뉴 열기/닫기 등 동작이 구현됨.

**4. MVC 구조와 웹 프레임워크**
MVC: Model(DB 관련 로직), View(화면 표시), Controller(M과 V의 연결 제어). 이 MVC 구조를 매번 새로 만드는 것은 비효율적이므로 웹 프레임워크를 사용. 프레임워크가 MVC 구조를 미리 만들어 놓음. 바이브코딩에서도 어떤 프레임워크를 사용할지 정하면 그 구조 안에서 개발.

**5. 데이터베이스 (DB)**
데이터를 저장하고 빠르게 조회하는 시스템. 엑셀과 동일한 테이블 구조. 텍스트 파일로 저장하는 것과의 차이: 1,000명 동시 조회 가능, 대용량 데이터 관리, 효율적 검색, 데이터 무결성 보장.

**6. 서버 프로그램**
서버 PC를 서버로 사용하게 해주는 프로그램. 설치하지 않으면 일반 PC. 대표적으로 Apache, Nginx. 사용자가 접속하면 웹페이지를 보내주는 역할.

### 네트워크 기초
- **IP 주소**: PC의 고유 주소 (우리 집 주소와 동일한 개념)
- **도메인**: IP 주소를 외우기 쉽게 만든 이름 (google.com, naver.com)
- **DNS (Domain Name Server)**: 도메인을 IP로 변환해주는 서버. 브라우저에서 google.com 입력 → DNS에서 IP 변환 → 해당 IP의 서버 PC로 접속 → 웹페이지 수신. 이 모든 과정이 순식간에 이루어짐

### JavaScript와 언어 발전사
- **JavaScript ≠ Java**: "핫도그가 도그의 업그레이드 버전이 아닌 것처럼" 이름만 비슷할 뿐 완전히 다른 언어. JavaScript는 클라이언트 언어, Java는 서버 언어
- **JavaScript의 범용성**: 웹 전 분야 사용, 게임도 가능, Node.js로 서버도 가능
- **언어 발전사**: 어셈블리(0과 1) → C(대중화 혁명) → 웹 언어(Java, PHP) → 파이썬(더 쉽고 짧게) → 바이브코딩(사람의 말로 시키기)
- **바이브코딩 시대의 의미**: "사람을 위한 프로그래밍 언어의 발전은 무의미해졌다. 기본 개념을 바탕에 깔아두고 말로 프로그램을 만들면 된다"

### 왜 웹앱부터 만드는가
웹으로 만들면 윈도우 프로그램, 모바일 앱, 맥 앱, 게임으로 변환 가능. HTML/CSS/JavaScript는 기본. 웹이 확장성과 호환성이 가장 좋음. 키오스크나 테이블 오더 프로그램도 웹으로 만들어짐.

### 백엔드 6가지

**1. 라우트**
URL 경로에 따라 페이지를 연결해 주는 기능. google.com으로 접속하면 메인 페이지, google.com/mail이면 메일 페이지. 슬래시 뒤의 경로에 따라 다른 페이지를 보여줌. 프레임워크가 기본 탑재.

**2. 웹 프레임워크**
자주 쓰는 구조와 기능을 미리 만들어 놓은 도구.
- Python 기반: Django, Flask, FastAPI (최근 FastAPI 인기)
- Node.js 기반: Express, Next.js
- Java 기반: Spring
- PHP 기반: Laravel, CodeIgniter

**3. API (Application Programming Interface)**
프로그램 간 소통 창구. "키오스크로 비유하면, 키오스크 화면은 사용자가 음식을 고르는 창구(API). 주방(서버)은 화면 뒤에서 실제 조리. 사용자는 주방 내부를 몰라도 화면(API)을 통해 음식을 주문할 수 있음"

**4. 배포**
만든 웹서비스를 인터넷에 공개. 로컬에서만 보이는 것을 전 세계에서 접속 가능하게 만드는 것.

**5. 서버리스와 직접 서버 운영**
서버리스: 직접 서버 관리 없이 클라우드 서비스 이용. 직접 운영: 서버 구매/설정/유지보수 직접.

**6. 깃허브와 호스팅**
GitHub: 코드 저장소 + 협업 플랫폼 + 버전 관리. 호스팅: 웹서비스를 인터넷에 올리는 서비스 (Vercel, AWS, Netlify 등).

### AI 관련 6가지
- **LLM**: 대형 언어 모델. AI의 뇌
- **프롬프트 엔지니어링**: AI에게 잘 물어보는 기술. 질문의 품질 = 답변의 품질
- **컨텍스트와 토큰**: 컨텍스트는 AI가 기억하는 대화 범위. 토큰은 글자 수 단위. 컨텍스트가 길어지면 AI가 앞을 잊음
- **컨텍스트 엔지니어링**: AI가 잊지 않도록 컨텍스트를 관리하는 기술 (치트키 4번과 연결)
- **할루시네이션**: AI가 자신감 넘치게 거짓말. 없는 라이브러리 추천, 틀린 코드를 정답처럼 제시 (치트키 13번과 연결)
- **AI 에이전트와 MCP**: 에이전트는 자율적으로 행동하는 AI. MCP는 AI가 외부 도구에 연결하는 규격
