# 지식 저장소 진화

## 정의
개인 지식 관리 도구의 역사적 진화. 종이 → Zettelkasten → PC/검색 → Evernote/Second Brain → Obsidian/양방향링크 → LLM Wiki로 이어지는 패러다임 전환.

## 핵심 원칙
각 단계는 **검색 비용**과 **유지보수 비용**의 문제를 해결하며 진화:
1. **종이**: 기록 자체가 목적
2. **Zettelkasten (1960s)**: 지식 연결 구조 도입 — "어떻게 찾을 것인가"의 시작
3. **PC 보급 (1990s)**: 기록 중심 → 검색 중심 전환
4. **클라우드 (2000s)**: Evernote, "Second Brain" — 정보 수집 폭발
5. **Obsidian (2020s)**: 양방향 링크, 지식 네트워크 — Zettelkasten의 디지털화
6. **LLM Wiki (현재)**: 검색(RAG) → 지식 재구성. "누가 유지보수하는가"의 해답 = LLM

## 적용 사례
- **Karpathy의 LLM Wiki**: 코드보다 개인 지식 저장소 구축에 더 많은 토큰 사용
- **이 위키 (0LLMWiki)**: LLM Wiki 패턴의 실천적 구현

## 다음 단계: 프로그래밍 가능한 Markdown
7. **Turing-complete Markdown (Quarkdown)**: 지식 저장소의 출력이 "정적 문서"에서 "실행 가능한 프로그램"으로 진화. 하나의 소스에서 책/논문/슬라이드/위키를 모두 생성. LLM이 생성한 지식을 다양한 포맷으로 자동 변환.
